# Design Principles

- Make playlist review clear before making generation flashy.
- Show accepted and rejected outcomes explicitly.
- Keep constraints visible and understandable.
- Prefer focused controls over broad configuration surfaces.
- Preserve keyboard access and readable responsive layouts.
- Reuse existing components before adding new UI patterns.
- Keep chat supportive, not dominant.
- Use CutList brand texture deliberately: The Curator, amber/green/cyan accents, and concise music-aware language are welcome when they support orientation.
- Keep workflow copy literal. Brand personality must not obscure controls, statuses, errors, verification, constraints, or destructive actions.
- Treat the Curator Console as shared input for playlist-change requests and review-focused text, not only track generation.

## Avoid

- Marketing-style landing pages inside the app.
- UI that implies account sync or streaming upload.
- Hidden state changes without visible feedback.
- Dense decoration that competes with track review.
- Components that duplicate existing playlist mutation logic.
- Roleplay-heavy labels that force users to translate the interface before acting.
